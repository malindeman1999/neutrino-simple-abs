Detuned Quadrature Mixing

Why TLS can appear in amplitude readout when the resonator is detuned.

Home Theory M Matrix Noise Sources Detuned Mixing TLS + Johnson Signal Processing

Short Answer

Yes, this is physical and expected: with nonzero detuning, phase-like and amplitude-like outputs are mixed by the linear response matrix.

So a TLS source that is introduced in the phase-like source vector can produce a nonzero amplitude-channel output.

What Changes Off Resonance

On resonance, quadratures align more closely with reactive (phase) and dissipative (amplitude) directions.

With detuning \(x\neq 0\), those directions rotate/mix. In matrix form:

\[ Y(\omega)=M^{-1}(\omega)\,N(\omega), \]

and off-diagonal couplings in \(M\) (including detuning-dependent terms) make the amplitude output depend on phase-like source components.

See Theory and M Matrix for the full coefficient definitions.

Does This Mean TLS Does Work?

No. Observing TLS in amplitude readout under detuning does not by itself imply TLS is supplying net dissipative power.

It primarily reflects readout-basis projection: a reactive fluctuation source can project onto the measured amplitude coordinate when the operating point is detuned.

Connection to the Python Model

In code, TLS is injected as a phase-like source vector and propagated through \(M^{-1}\):

  • Sensor.n_tls_phi() defines the TLS source vector.
  • Sensor._propagate_noise_vector() applies matrix propagation.
  • The GUI computes both phase and amplitude channel outputs versus frequency and converts each to NEP using channel-specific responsivity.

Practical implication: in amplitude-readout mode, a TLS trace can be nonzero even when TLS is modeled as phase-like at injection.
